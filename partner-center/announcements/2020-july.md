## <a name="changes-coming-to-partner-center-apis"></a><a name="9"></a>Změny přicházející do rozhraní API partnerského centra

### <a name="categories"></a>Kategorie

- Datum: 28. 7. 2020
- Možnosti

### <a name="impacted-audience"></a>Ovlivněná cílová skupina

Partneři s přímým vyúčtováním a nepřímí poskytovatelé účastnící se programu CSP, kteří používají Partnerské centrum API. 

### <a name="summary"></a>Souhrn

Tým Partnerské centrum vyhodnotil zveřejněná Partnerské centrum API. Vidíme, že některá z těchto rozhraní API jsou zastaralá a už se nepodporují. Do 31. srpna 2020 budou vyřazeny.

### <a name="details"></a>Podrobnosti

Tým csp Partnerské centrum identifikoval několik rozhraní API, která máme v plánu vyřadit do konce 31. srpna 2020. Tato rozhraní API představují rozhraní, která byla navržena před několika lety, ale řada z nich už není funkční. Tým v posledním kalendářním čtvrtletí 2020 odebírá odkazy na tyto koncové body v dalším vydání sady SDK. Níže jsou uvedená rozhraní API, která se mají vyřazena z provozu. Partneři by měli zajistit, aby se na tyto identifikátory URI odchýlili od budoucna.

**Žádosti o partnerskou službu** – Tyto koncové body partnerovi umožnily získat řetězce témat podpory a pak programově odesílat žádosti o partnerské služby. V průběhu času se toto rozhraní API stalo nestabilním a nepodporuje se. Partneři jsou přesměrováni do uživatelského rozhraní Partnerského centra, kde mohou vytvářet lístky podpory pro partnery. Uživatelské prostředí nabízí partnerům další informace při vytváření případů podpory, jako jsou doporučené kroky a dokumenty pro oblast, se které mají problémy. Tým Partnerského centra také vylepšoval prostředí pro vytváření lístků partnerů tím, že přizpůsobil formuláře žádostí o služby tak, aby zachytávání informací specifických pro oblast problému. Informace shromážděné prostřednictvím online formulářů pomáhají technikům podpory rychleji a přesněji vyřešit vaše problémy. Na tyto identifikátory URI by se nemělo odkazovat ani na ně spoléhat a do konce srpna 2020 bude vyřazeno z provozu. Uvědomte si, že tyto změny  mají vliv pouze na témata získání podpory a vytváření metod žádostí o partnerskou službu. Všechna ostatní rozhraní API související s podporou by měla dál fungovat podle očekávání.

- {base-URL}/v1/servicerequests/supporttopics
- {base-URL}/v1/servicerequests/{agent-locale}

**Office služby** – Tyto koncové body byly navrženy tak, aby partnerovi umožnily získat agregované zobrazení incidentů a výpadků služeb napříč všemi službami partnera. Data, která tyto koncové body vrací, jsou ale zastaralá a nejsou užitečná. Vyřazení těchto koncových bodů z provozu bude do konce července. Tým partnerského centra bude dále vyhodnocovat scénáře incidentů a stavu služeb pro všechny zákazníky a hledat příležitost k přidání hodnoty v této oblasti. Odkazy na tyto identifikátory URI by se měly odebrat.

- {base-URL}/v1/serviceincidents/office/count
- {base-URL}/v1/serviceincidents/office

Partneři mohou pomocí [rozhraní API Office 365 Service Communications](/office/office-365-management-api/office-365-service-communications-api-reference) získat aktuální stav služeb zákazníkům.

**TopProduct** – tento koncový bod byl vytvořen jako důkaz konceptu, který nikdy nebyl určený pro obecnou dostupnost. Byla navržena tak, aby partnerům umožnila spravovat nejlepší nabídky, které by se vykreslují v Partnerské centrum katalogu. Toto rozhraní API ale nebylo nikdy plně vyvinuto a dokončeno a do 31. července 2020 bude zakázané. Partneři by na něj měli odebrat všechny odkazy.

- {base-URL}/v1/preferences/catalog/products

### <a name="next-steps"></a>Další kroky

- Partneři by měli zajistit, že jejich řešení už nebudou odkazovat na výše uvedená rozhraní API.
- Cloud Solution Provider partneři mohou navštívit web [SDK pro Partnerské centrum a rozhraní API Yammer a](https://www.yammer.com/cloudpartnercommunity/#/Threads/show?threadId=786951767490560) klást otázky v případě potřeby dalších objasnění.

## <a name="sdk-release-on-net-standard-v1161-preview"></a><a name="8"></a>Vydání sady SDK na .NET Standard (verze 1.16.1-preview)

### <a name="categories"></a>Kategorie

- Datum: 21. 7. 2020
- Možnosti

### <a name="impacted-audience"></a>Ovlivněná cílová skupina

Partneři s přímým vyúčtováním a nepřímí poskytovatelé účastnící se programu CSP, kteří používají Partnerské centrum .NET SDK. 

### <a name="summary"></a>Souhrn

Od 21. července 2020 mohou partneři začít stahovat verzi Preview sady **.NET Standard SDK microsoft Partnerské centrum** .NET SDK v1.16.1-preview a také aktualizované ukázky veřejného **SDK pro Partnerské centrum GitHub**.

### <a name="details"></a>Podrobnosti

Partneři s přímým vyúčtováním a nepřímí poskytovatelé účastnící se programu CSP mají nárok na stažení nejnovější verze Preview sady .NET Standard SDK. Migraci aplikace Microsoftu z SDK pro Partnerské centrum .NET Framework na .NET Standard 2.0. Díky těmto změnám bude sada SDK kompatibilní se stávající aplikací pomocí .NET Framework verze 4.6.1 a vyšší. Tento .NET Standard bude podporovat .NET Core 2.0 a vyšší.

Od partnerů, kteří požadují sadu SDK, v .NET Standard. Aktuální verze sady SDK nutí partnery používat aplikace napsané v ASP.NET Frameworku, což je pomalé a nákladné porovnání s aplikacemi .NET Core. Migrace na .NET Standard proto může pomoct vyřešit problémy s výkonem nebo kompatibilitou s .NET Core.
 
**Nejzajímavější body**
- Žádné další verze od Microsoftu pro .NET Framework 4.8, protože se jedná o poslední na řádku.
- Omezuje budoucí vývoj, protože většina knihoven bude vymáněná pro .NET Core nebo .NET Standard, který nemusí podporovat aktuální verzi architektury.
- Vzhledem ke své jednoduché povaze je použití .NET Core s kontejnery výhodné.
- Vysoce výkonný serverový modul runtime pro Windows Server a Linux činí .NET Core vysoce výkonným webovým rozhraním.

řídicí panel Přehledy v partnerském centru poskytuje jednotný, 360ý pohled na klíčové ukazatele výkonu vaší partnerské organizace v cloudových produktech, jako jsou Office 365, Azure, Dynamics 365 a licenční modely, jako je Cloud Solution Provider (CSP) a smlouva Enterprise (EA). Vám budou doručovány inteligentní přehledy, abyste mohli jasně porozumět vašemu podnikání společnosti Microsoft a mohli svá rozhodnutí řídit daty s jistotou.   

pokud se chcete podívat na tento řídicí panel, vyberte **Přehledy** z navigační nabídky centra partnera. Tento řídicí panel používá stejná základní data jako aktuální Nástroj pro vytváření sestav MyInsights, ale nabízí bohatší vizuály a další možnosti vytváření sestav přehledné a klíčové ukazatele výkonu zahrnující zákazníky, předplatná, využití licencí, kompetence a výhody. vraťte se často, protože budeme dál přinášejíme nové sestavy a možnosti na řídicí panel Přehledy.

Neposkytovatelé nepřímých prodejců a kvalifikovaný zprostředkovatel CSP 

### <a name="details"></a>Podrobnosti

V Microsoftu se zavazujeme podpořit každou organizaci svou cestou digitální transformace. Díky tomu nestačí jenom v technologii, ale také prozkoumáním koncových zákazníků a partnerů, od získání naší technologie po jejich prostředí podpory. Díky této práci jsme identifikovali příležitost k transformaci dodržovali – díky investicím do naší obchodní platformy.  Také jsme zjednodušili licenční programy, a proto jsme vyvinuli nové prostředí pro obchod, abychom mohli vylepšit způsob, jakým naši partnerům poskytujeme více příležitostí. 

vzhledem k tomu, že jsme vylepšili naše možnosti obchodování a poskytovali zákazníkům a partnerům lepší možnosti nákupu, rozšíříme portfolio nabídek, aby zahrnovalo dostupnost trvalé licence softwaru v rámci Cloud Solution Provider programu (CSP). Přidání licencí softwaru v rámci CSP umožňuje zúčastněným partnerům soustředit se na poskytování různých řešení a spravovaných služeb zákazníkům. Umožňuje také převod zákazníků do cloudu a poskytuje partnerům flexibilitu, kterou potřebují pro hybridní cloudová prostředí.   

Od 1. července jsme tuto funkci provedli prostřednictvím sady kvalifikovaných nepřímých zprostředkovatelů a jejich nepřímými prodejci, kteří budou moci v programu CSP získat licence pro místní software se zákazníky. Kvalifikovaní partneři spravují Microsoft a musí být aktivními poskytovateli CSP a autorizovanými distributory Microsoftu pro Open License program, který je v současnosti podporovaný pro prodej softwaru v rámci CSP. Od ledna 2021 bude Microsoft tuto funkci dál škálovat a umožní všem partnerům s přímým účtováním CSP, nepřímým poskytovatelům a nepřímým prodejcům prodávat licence na software v rámci programu CSP.
