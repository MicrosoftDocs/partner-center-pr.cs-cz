## <a name="software-in-microsoft-china-corporation-limited-mccl-cloud-solution-provider-csp-and-open-license-change"></a><a name="9"></a>Software v Microsoft China Corporation Limited (MCCL) Cloud Solution Provider (CSP) a Open License změny

### <a name="categories"></a>Kategorie

- Datum: 13. 7. 2021
- Možnosti
 
### <a name="summary"></a>Souhrn

Po aktualizaci z [6. května 2021](https://partner.microsoft.com/resources/detail/csp-perpetual-software-coming-to-china-pdf) vás chceme u našeho průběhu aktualizovat klíčovými oznámeními o cílovém programu Software in Microsoft China Corporation Limited (MCCL) Cloud Solution Provider (CSP) a změnách programu Open.

### <a name="impacted-audience"></a>Ovlivněná cílová skupina

Partneři v programu MCCL CSP

### <a name="details"></a>Podrobnosti

#### <a name="software-in-mccl-csp-general-availability-date"></a>Datum obecné dostupnosti programu MCCL CSP 

Cílení na první týden v srpnu 2021 budou moct partneři zaregistrovaní v programu MCCL CSP prodávat časově neomezené softwarové licence pro místní nasazení prostřednictvím programu MCCL CSP. Zákazníci tak budou mít flexibilitu licencování, kterou potřebují v rámci digitální transformace.

#### <a name="open-license-program-retirement"></a>Open License program vyřazení

Od 1. ledna 2022 už zákazníci nebudou moct kupovat nové nebo obnovovat softwarové licence prostřednictvím programu Microsoft Open License. Do té doby mohou přes program Open License nadále nakupovat všichni zákazníci.

#### <a name="open-license-program-price-change-for-government"></a>Open License cen pro státní instituce v programu

Od července 2021 zákazníci státní správy uvidí různé zvýšení cen místních SKU programu Open License, jako jsou licence (L), Software Assurance (SA) a licence Microsoftu a Software Assurance (L&SA). Jedná se o pokračování práce, která začala v roce 2018, aby se nastavení cen pro státní instituce srovnaly s novým komerčním prostředím. Cena Open License státní správy se pak bude rovnat nejnižší komerční ceně, což vytvoří paritu s cenami nového komerčního prostředí v programu CSP a usnadní zákazníkům ze státní správy porovnávání řešení. Další informace najdete v nejčastějších [](https://aka.ms/openlicensegovpricefaq) dotazech k cenám Open License government a v balíčku [L100.](https://aka.ms/openlicensegovpriceL100)

#### <a name="sa-purchases"></a>Nákupy SA

Výhody SA budou pokračovat až do vypršení platnosti období SA, a to i v případě, že platnost vyprší po 1. lednu 2022. Program Microsoft Open Value (OV) se doporučuje pro budoucí nákupy, které zahrnují SA.

>[!NOTE]
>Programy OV ani Open Value Subscription se neplánovat.

Partneři zaregistrovaní nebo se brzy zaregistrují do programu MCCL CSP by si měli zkontrolovat následující materiály a sdílet je s příslušnými zúčastněnými stranami v rámci své organizace:

Podívejte se také na související Open License programu změn připravenosti na webu [partnera Microsoftu.](https://partner.microsoft.com/resources/collection/csp-open-evolution-to-a-better-experience#/)

## <a name="readiness-changes-to-the-cloud-solution-provider-csp-customer-address-validation-api-to-go-live-on-july-13-2021"></a><a name="8"></a>Připravenost: Změny rozhraní API pro ověřování Cloud Solution Provider (CSP) zákazníka budou živé 13. července 2021.

### <a name="summary"></a>Souhrn

Změny rozhraní API pro ověření adresy budou živé od 13. července 2021.

### <a name="categories"></a>Kategorie

- Datum: 9. 7. 2021
- Připravenost

### <a name="impacted-audience"></a>Ovlivněná cílová skupina

Cloud Solution Provider adresy partnerů s přímým vyúčtováním (CSP) a nepřímých poskytovatelů, kteří vytvářejí nové nebo aktualizují stávající zákazníky

### <a name="details"></a>Podrobnosti

Microsoft běží na vztahu důvěryhodnosti. Zavázali jsme se poskytovat vyhovující, bezpečnou a zabezpečenou metodu ověřování adres zákazníků pro transakce s předplatným zákazníků v programu CSP. Od 13. července 2021 budou změny rozhraní API pro ověření adresy živé.
 
Tyto změny mají vliv pouze na rozhraní API pro ověření adresy. Na rozhraní API pro vytvoření zákazníka a aktualizaci fakturačního profilu to nebude mít vliv. V současné době se navrhovaná adresa nemusí používat s rozhraním API pro vytvoření zákazníka, ale důrazně se doporučuje.

Odpověď vrátí jednu z následujících stavových zpráv:

| Status | Popis | Počet vrácených navrhovaných adres |
|------------|---------------|------------------|
| Ověřená expedice  | Adresa je ověřená a je možné ji poslat na adresu .  | Jednoduché  |
| Ověřené  | Adresa je ověřená.  | Jednoduché  |
| Požadovaná interakce  | Navrhované adresy se výrazně změnily a je potřeba potvrzovat uživatele.  | Jednoduché  |
| Částečná ulice  | Ulice v adrese je částečná a potřebuje další informace.  | Více – maximálně 3  |
| Částečně místní  | Dané prostory (číslo budovy, číslo sady a další) jsou částečné a potřebují další informace.  | Více – maximálně 3  |
| Několik  | Adresa obsahuje několik polí, která částečně zahrnují i část ulice a částečnou část místně.  | Více – maximálně 3  |
| Žádná  | Adresa je nesprávná.  | Žádná  |
| Není ověřeno.  | Adresu nebylo možné odeslat prostřednictvím procesu ověřování.   | Žádná  |

Po odeslané adrese, která se má ověřit prostřednictvím rozhraní API pro ověření adresy, se vrátí následující schéma odpovědi:

```csharp

// <summary>
/// Object represents the address validation response.
/// </summary>

public class AddressValidationResponse
{
   /// <summary>
   /// Gets or sets the original address
   /// </summary>
   /// <value>
   /// Original Address
   /// </value>
   public Address OriginalAddress { get; set; }

   /// <summary>
   /// Gets or sets the suggested addresses
   /// </summary>
   /// <value>
   /// Suggested Addresses
   /// </value>
   public List<Address> SuggestedAddresses { get; set; }

   /// <summary>
   /// Gets or sets the validation status
   /// </summary>
   /// <value>
   /// Status
   /// </value>
   public string Status { get; set; }

   /// <summary>
   /// Gets or sets the validation message
   /// </summary>
   /// <value>
   /// Validation Message
   /// </value>
   public string ValidationMessage { get; set; }

```

Všimněte si, USA v odpovědi vrátí další čtyřmístné přípony pro řádek PSČ, pokud zadáte jenom pět číslic PSČ.
Následuje ukázková odpověď:

```C#

"suggestedAddress": [

"country": "US",

], 
"originalAddress": { 
"country": "US", 
"city": "Redmond", 
"addressLine1": "1 Micro Way", 
"postalCode": "98052" 
}, 
"status": "InteractionRequired", 
"validationMessage": "Address field invalid for property: ‘Street’" 
}
```

## <a name="keep-your-microsoft-certifications-active-with-help-from-partner-center"></a><a name="6"></a>Udržování aktivních certifikátů Microsoftu s využitím partnerského centra

### <a name="categories"></a>Kategorie

- Datum: 2021-07-09
- Rozšiřte svou firmu

### <a name="impacted-audience"></a>Ovlivněná cílová skupina

Správci partnerů a globální Správci spravující stav a obnovení certifikace Microsoftu

### <a name="details"></a>Podrobnosti

Vaše certifikace Microsoftu je teď teď jednodušší, a to s využitím partnerského centra.

na řídicím panelu partnerského centra jste vždycky viděli certifikace vaší organizace, ale teď můžete také zobrazit jejich datum vypršení platnosti v sestavě dovednosti uživatelů i v sestavě školení Přehledy partnera. Tyto aktualizace zavedou kontrolu nad procesem obnovování certifikace ve vaší organizaci. Přečtěte si, kde najdete informace o tom, jak najít aktualizace sestav a jak vám budou mít výhody.

V části kompetence na řídicím panelu partnerského centra najdete sestavu kompetence. při exportu dat do Excel se zobrazí sestava odbornosti uživatele, která poskytuje seznam získaných výsledků (například certifikace a zkoušky), které vám opravňují k kompetenci. Na konci této sestavy je nový sloupec s názvem "datum vypršení platnosti dovednosti", kde najdete data vypršení platnosti pro vaše zaměstnance a specializace na základě rolí.  

v části dovednosti organizace na řídicím panelu partnerského centra najdete zprávu o školení pro Přehledy partnera. když exportujete data této sestavy do Excel, najdete nový sloupec s datem vypršení platnosti, který obsahuje data vypršení platnosti pro certifikace vašeho týmu.

Tyto aktualizace vám pomůžou rychle provést akci a připomenout si své zaměstnance, aby si obnovili své certifikace a zachovali jejich přihlašovací údaje i stav programu MPN aktivní.

Jak funguje nový proces obnovování certifikátů?  

- Místo přechodu na Centrum testování a placení na obnovu certifikačních zkoušek budou vaši zaměstnanci navštěvovat Microsoft Learn a předávat bezplatné posouzení obnovy certifikace, které je testuje na nejnovější technologické aktualizace.
- S tím, jak se data vypršení platnosti certifikace blíží, dostanou e-mailová oznámení od Microsoftu a nasměrují je na posouzení obnovení certifikátů na Microsoft Learn.
- Online postupy učení obsahují studijní materiály pro přípravu na posouzení, které se můžou pokusit tolikrát, kolikrát potřebují.
- Předání posouzení prodlouží platnost certifikace po dobu jednoho roku.
- Tento proces se opakuje každý rok, přičemž bude zajištěna aktuálnost dovedností.

V souladu s tímto novým ročním procesem obnovování mění společnost Microsoft také časový rámec platnosti pro role a specializace založené na rolích od dvou let po jeden rok. Od 30. června 2021 budou všechny vytvořené role založené na rolích a specializované certifikace platit po dobu jednoho roku.

Upozorňujeme, že základní a starší verze certifikačních autorit Microsoftu jsou vyloučené a neovlivní nový proces.

### <a name="next-steps"></a>Další kroky

1. přečtěte si datum vypršení platnosti pro certifikace týmu: projděte si zprávu o dovednostech uživatelů nebo školicím centru Přehledy školení a identifikujte datum vypršení platnosti certifikace.
2. Proveďte akci: požádejte tým, aby obnovil své certifikace od Microsoftu před datem vypršení platnosti.

## <a name="dynamics-365-cloud-migration-promotion-and-retirement-of-discounted-offers"></a><a name="1"></a>Dynamics 365 Migrace do cloudu povýšení a vyřazení zlevněných nabídek

### <a name="categories"></a>Kategorie

- Datum: 1. 7. 2021
- Možnosti

### <a name="summary"></a>Souhrn

Microsoft s radostí oznamuje spuštění propagační akce Dynamics 365 Migrace do cloudu 1. srpna 2021.

### <a name="impacted-audience"></a>Ovlivněná cílová skupina

Cloud Solution Provider (CSP), místní partneři Dynamics a partneři LPS

### <a name="details"></a>Podrobnosti

Prostřednictvím programu CSP mohou partneři digitálně transformovat firmy svých zákazníků pomocí špičkových cloudových řešení. V rámci tohoto úsilí 31. července 2021 vyřazení stávajících zlevněných SKU (viz karta Vyřazené skladové ceny v Dynamics 365 Migrace do cloudu a vyřazené soubory SKU [Excel)](https://partner.microsoft.com/resources/detail/dynamics-365-cloud-promotion-retired-skus-xls)vyřazené z provozu a jejich nahrazení novými skladovoumi nabídkami Dynamics 365 Migrace do cloudu.

Microsoft s radostí oznamuje spuštění propagační akce Dynamics 365 Migrace do cloudu 1. srpna 2021. Toto nové povýšení poskytuje zlevněné aplikace Dynamics 365 online služby, které pohánět migraci zákazníků do cloudu. Podporuje také zákazníky, kteří zahájili cestu digitální transformace v rámci nabídek starší verze migrace, aby přechází na nové ceny povýšení.

#### <a name="dynamics-365-cloud-migration-promotion"></a>Povýšení dynamics 365 Migrace do cloudu nasazení

**Výhody**

Propagační Migrace do cloudu nabízí zákazníkům následující výhody:  

- 40% sleva pro Dynamics 365 Online Services s tříletým předplatným, která poskytuje předvídatelnou cenu během migrace do cloudu.

- Duální přístupová práva (souběžné místní a cloudové použití), podpora přístupu uživatelů před tím, než se místní řešení dokončí, po dokončení migrace do cloudu.

- Podporuje postupné migrace do cloudu, což organizacím umožňuje přidávat nové uživatele během období povýšení.

**Podmínky**

Licenční program: CSP

Produkt: Povýšení dynamics 365 Migrace do cloudu nasazení

Propagační sleva: 40 %

Doba trvání povýšení: 1. srpna 2021 až 30. června 2023

Období předplatného: Tříletá předplatná; propagační cena není obnovitelné.

Licenční požadavky:

- Všechny místní licence Dynamics (Dynamics AX, CRM, partner pro státní správu, NAV, licence předplatného a další) s aktivním plánem Dynamics Customer Services a podpory (například Plán vylepšení nebo plán Advantage) nebo Software Assurance (SA) a prodloužení platnosti do Dynamics 365 online služby.

- Všechny dynamics 365 online služby s aktivními předplatným pro Dynamics 365 ze SA nebo Dynamics Price List, doplňku Dynamics 365 Cloud, kvalifikované nabídky Dynamics 365 nebo nabídky Dynamics 365 Migrace do cloudu.

- Toto povýšení se neaktivním místním zákazníkům Dynamics neaktivním zákazníkům v cloudu. (Další informace najdete [v online služby zabezpečení.)](https://www.microsoft.com/licensing/terms/productoffering/MicrosoftDynamics365Services/EAEAS)
